Java之枚举

Aug 21, 2016 | JAVA

引入枚举的背景

场景

为彩虹描绘出其中颜色，通常可以通过定义常量的方式来实现

但这种方式会存在以下这几个问题：

类型不安全

由于颜色常量的对应值是整数型，所以程序执行过程中很有可能给颜色变量传入一个任意的整数值，导致出现错误。

类型无指意性

由于颜色变量值仅仅是一些无包含任何含义的整数值，如果在运行期调试时候，你就会发现日志中有很多魔术数字，但除了程序员本身，其他人很难明白其中奥秘。

枚举类型

为了改进Java语言在这方面的不足，JDK1.5之后，增加了枚举类型。枚举是限定有限种可能值的一种手段，使用枚举可以降低程序出错的几率，并可以提高代码的可读性和可维护性

枚举定义

常规用法

枚举提供了几个方法供开发者在需要时调用。

方法 描述
vlaues() 返回当前枚举中所有枚举项的集合
oridinal() 返回枚举项在枚举对象中的序号
name() 返回枚举项的名称，与toString()效果相同
getDeclaringClass() 取得当前枚举值所在类的完整名称
valueOf(“xx”) 返回名称为xx的枚举项

枚举集合

集合对象 描述
EnumSet EnumSet用于保存枚举项的集合，在枚举项本身并不互斥的情况下特别有用。例如，一个应用系统通常会有多种角色，而某些人在系统中可能承担不止一种角色。如果这些角色是通过枚举定义的，则此时EnumSet将可以将多种角色保存在一起，标识特定用户承担的全部角色
EnumMap EnumMap是必须使用枚举项作为其KEY的Map，其法与普通Map类似
