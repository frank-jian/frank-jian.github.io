Hexo免输入密码部署到Github

发表于 2016-04-14 | 分类于 Hexo |

在使用hexo deploy命令部署hexo到github时，每次都要输入用户名和密码，这种重复机械的流程，让人感觉很烦躁。以下是介绍如何解决这个问题：

3、设置好这些之后，当你再次部署时，就不用再输入用户名和密码了。

2、生成SSH密钥

  • 查看是否已经有了ssh密钥

如果没有密钥则不会有此文件夹，有则备份删除

  • 生成密码

按3个回车，密码为空，得到两个文件id_rsa和id_rsa.pub

3、在github上添加ssh密钥，这里添加的是id_rsa.pub里面的公钥

4、修改_config.yml中deploy的参数，部署仓库以SSH的方式发布不需输入密码，反之如果以HTTPS的方式发布则需要密码。
