前言

总是在写重复代码，有没有觉得很煎熬？对代码多点挑剔，你会想法设法尽量去避免写重复代码。学习和借鉴优秀源码的设计思想，用于实际项目中，让你的代码变得更加优雅。

实战

1. Map接口

解读Map源码，学到了以下几点：

  1. 泛型的使用
  2. default声明方法体的方法；
  3. 接口中可声明静态方法
  4. 接口中嵌套接口

2. MapAbstract抽象方法

解读MapAbstract源码，学习到以下几点：

  1. 抽象类中，至少要有一个抽象方法。
  2. 抽象类，不能有接口，除了抽象方法，方法必须有方法体。
  3. 抽象类中，可以声明静态类。
